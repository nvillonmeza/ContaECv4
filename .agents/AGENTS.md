# Agent Rules for ContaECv4

## Actualización de Handoff (handoff.md)
Para cada participación o sesión de trabajo sobre el código, el agente DEBE actualizar obligatoriamente el archivo `handoff.md` ubicado en la raíz del proyecto.

Al actualizar `handoff.md`, debes incluir o actualizar la siguiente información:
- **Objetivo**: Qué se busca lograr en la sesión.
- **Estado actual**: Cómo se encuentra el proyecto en este momento.
- **Archivos en los que estás trabajando**: Lista de archivos que se están modificando.
- **Qué has cambiado**: Detalle de las modificaciones realizadas.
- **Qué has intentado**: Enfoques probados.
- **Qué ha fallado**: Errores encontrados o cosas que no funcionaron.
- **Qué planeas hacer después**: Siguientes pasos (Plan de Próximos Pasos).

**REGLA CRÍTICA:** TIENES PROHIBIDO borrar información previa de `handoff.md`. Solo debes agregar, complementar o actualizar las secciones existentes manteniendo el historial íntegro.
